Your policy sandbox is ready

Policy Maker is a sandbox for designing and testing AI agent governance rules — the logic that decides what actions an autonomous agent is and isn't allowed to take. There's no live agent connected here: this is purely a place to explore, experiment, and understand how policy-based access control works before you deploy it for real.

How it works — three steps
✦
Create Rules Step 1
Describe what you want to allow or block in plain English. The AI assistant translates your intent into a structured YAML rule and proposes it for your review. Accept it, edit it, or discard it — you stay in control.
Example: "Block any shell command that runs rm" → deny rule, tool: bash, program: rm
▷
Test Action Step 2
Test any hypothetical action against your current rule set. Fill in the tool, program, and path an agent would use, then hit Evaluate. You'll see the verdict — allow, deny, or pending — and a full trace showing every rule that was checked and why it matched or didn't.
Example: tool=bash · program=git · path=/prod/repo → which rule fires?
📋
Policy Rules Step 3
View and manage your full rule set. Add, edit, or delete rules directly. Run the policy analyzer to catch problems: shadowed rules that can never fire, conflicting rules at the same priority, or overly broad allows that permit more than intended. Import a YAML file to load a starter set, or export your rules to use elsewhere.
Tip: rules are evaluated highest priority first — the first match wins.
Key concepts
result: allow / deny / pending
What happens when a rule matches. allow permits the action, deny blocks it, pending queues it for human approval.
priority
Rules are sorted highest-first. The first rule whose conditions all match is the one that fires — lower rules are never reached.
match conditions
All conditions on a rule are ANDed. Leave a field blank and it matches anything. An empty match block is a catch-all that matches every request.
tool / program / path
tool is the agent capability (bash, read, write…). program is the executable within bash. path supports * and ** glob patterns.
shadow
A higher-priority rule whose conditions are a subset of a lower one makes that lower rule unreachable — it is shadowed. The analyzer flags these.
skill prompt
The system prompt used by the AI Agent Rule Maker and Analysis assistant. You can download it, customise it for your domain, and upload your own version.
